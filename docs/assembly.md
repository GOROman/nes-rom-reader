# 組み立てガイド(初心者向け・ステップバイステップ)

[English](assembly.en.md) | **日本語**

nes-rom-reader v0.1 基板を、はんだ付けから動作確認まで組み立てる手順です。
**表面実装(SMD)のはんだ付けは初めてでも作れます。** 所要時間の目安は 1〜2 時間。

![完成した基板](images/assembly/07-complete.jpg)

---

## ⚠️ 最初に読んでください(v0.1 の既知の問題)

**v0.1 基板は M5Stamp S3 がそのまま挿さりません。**
Stamp 装着部のフットプリント(穴の配置・サイズ)が実物と合っていませんでした。**設計上の不具合です。**

**回避策**: ピンヘッダで嵩上げして、配線を合わせて実装します(手順4で詳しく説明)。

![ピンヘッダで嵩上げ](images/assembly/06-stamp-headers.jpg)

> この問題は v0.2 基板で修正予定です。作る前に必ずこの点を理解しておいてください。

## 1. 必要な工具

### 必須

| 工具 | 選び方・補足 |
|---|---|
| **はんだごて** | 温度調節付き(**280〜320℃**)が理想。こて先は**細めのC型かD型**。100均のものは温度が安定せず難しいです |
| **はんだ** | **Φ0.6mm** の**ヤニ入り(フラックス入り)**。鉛入り(Sn60/Pb40)の方が溶けやすく初心者向け |
| **フラックス** | **これが最重要**。ICのはんだ付け成功率が段違いに上がります。ペーストタイプかペンタイプ |
| **ピンセット** | 先の細いもの。チップ部品(1.6mm×0.8mm)をつまみます |
| **はんだ吸い取り線** | ブリッジ(はんだ橋)の修正用。**必ず用意**してください |
| **テスター(マルチメーター)** | 導通・ショート確認用。1000円台のもので十分 |

### あると格段に楽

| 工具 | 用途 |
|---|---|
| **フラックス洗浄剤** | 作業後にヤニを落とす。仕上がりがきれい&接触不良防止 |
| ルーペ / 拡大鏡 | ブリッジや部品の向きの確認。スマホのカメラでも代用可 |
| 基板固定具(ヘルピングハンド) | 基板が動かないよう固定 |
| こて先クリーナー | 金属たわしタイプが便利 |

![作業風景](images/assembly/01-start-soldering.jpg)

> 💡 **初心者の方へ**: 「フラックス」「はんだ吸い取り線」「温度調節付きこて」の3つを揃えるだけで、
> 難易度が劇的に下がります。ここをケチると苦戦します。

## 3. はんだ付け(背の低い部品から)

**低い部品 → 高い部品**の順に付けると、基板が安定して作業しやすくなります。

```
① チップ抵抗 R1, R2       ← いちばん低い
② チップコンデンサ C1–C9   (任意)
③ ポリヒューズ F1          (任意)
④ ロジックIC U2–U8
⑤ M5Stamp(嵩上げ実装)
⑥ FC 60ピンコネクタ J1     ← いちばん大きい
```

### ① チップ抵抗(R1, R2)

![抵抗の実装](images/assembly/03-place-resistor.jpg)

**チップ部品の付け方(基本テクニック)**
1. パッドの**片側だけ**に、はんだを少しだけ盛る
2. ピンセットで部品を押さえながら、盛った側のはんだを**溶かして固定**
3. 反対側をはんだ付けする
4. 最初の側をもう一度溶かして整える

**見分け方**: 抵抗は表面に数字が印字されています。
- **R1 = `1002`**(10kΩ)
- **R2 = `2002`**(20kΩ)
- 極性(向き)はありません

### ② チップコンデンサ(C1–C9)※任意

**パスコンは、付けたければ付ける**程度で構いません(無くても動作します)。
付ける場合は抵抗と同じ要領です。**コンデンサには印字が無い**ので、
100nF(C1–C8)と 10µF(C9)を混ぜないよう、袋から出す前に区別してください。極性なし。

### ③ ポリヒューズ(F1)※任意

**お好みで。持っていなければ、F1 のパッド間を「ジャンパ線」で短絡**すれば動作します
(過電流保護が無くなるだけです)。極性なし。

### ④ ロジックIC(U2–U8)

![ICをはんだ付け](images/assembly/04-ics-soldered.jpg)

**⚠️ 向きが最重要です。** IC の**1番ピン側**(丸いくぼみ、または端の切り欠き)を、
基板シルクの**丸印・切り欠き側**に必ず合わせてください。逆向きだと動きません。

| リファレンス | 部品 | 役割 |
|---|---|---|
| U2, U3, U4, U5 | 74HC595 (SOIC-16) | アドレス生成(シフトレジスタ) |
| U6, U7 | 74LVC245 (SOIC-20) | データ読み出しバッファ |
| U8 | 74HCT541 (SOIC-20) | 制御線バッファ(**任意** → 下記) |

**ICのはんだ付け(ドラッグはんだ)**
1. **対角の2ピンだけ**を仮止めして位置を固定
2. ここで**向き・ズレを必ず確認**(この時点なら修正が簡単)
3. ピン全体に**フラックスをたっぷり**塗る
4. こて先に少量はんだをつけ、ピン列を**すーっと引く**
5. ブリッジしたら**はんだ吸い取り線**で吸う(フラックスを足すと吸いやすい)

> 💡 ブリッジしても慌てないでください。フラックス+吸い取り線でほぼ確実に直せます。

#### 🔧 74HCT541(U8)が無い場合 → ジャンパで代用

U8 は ESP32 の 3.3V 信号を 5V に変換するバッファですが、**無くても読み出しは動作します。**
U8 のパッド上で、以下のピン同士を**ジャンパ線(またはハンダブリッジ)で直結**してください。

```
2–18, 3–17, 4–16, 5–15, 6–14, 7–13, 8–12, 9–11
```

これで ESP32 の信号が直接 595・カートリッジへ届きます。

### ⑤ M5Stamp の実装(⚠️ v0.1 の要注意ポイント)

![Stampの嵩上げ実装](images/assembly/06-stamp-headers.jpg)

**v0.1 基板は Stamp がそのまま挿さりません。** 上の写真のように、
**ピンヘッダで嵩上げして**実装します。

1. ピンヘッダを基板側のパッド列にはんだ付け
2. その上に Stamp を載せ、各ピンを配線・はんだ付け
3. **USB-C コネクタが基板の外(下辺のノッチ側)を向く**ように配置

<details>
<summary>基板が期待するピン並び(照合用)</summary>

**左辺(17ピン、上→下)**
`G1, G2, G3, G4, G5, G6, G7, G8, G9, G10, GND, G11, 5V, G12, G13, G14, G15`

**右辺(11ピン、上→下)**
`3V3, G46, G43, G42, G44, G41, EN, G40, G0, G39, GND`
</details>

> ⚠️ 配線を間違えると動作しません。**テスターで数本の導通を確認**してから通電してください。

### ⑥ FC 60ピンスロットコネクタ(J1)

![基板の様子](images/assembly/05-board-progress.jpg)

1. 基板の**上辺**(シルクに `▼ FRONT` とある側)に挿す
2. まず**両端の2ピンだけ**はんだ付けし、コネクタが**基板に密着し、傾いていないか**確認
3. 問題なければ残りの58ピンをはんだ付け

> 💡 60ピンは数が多いので、**5〜10ピンごとにフラックスを塗り直す**ときれいに付きます。
> 終わったらルーペでブリッジがないか確認しましょう。

---

## 4. 通電前チェック(必ずやってください)

**Stamp に USB を挿す前に**、テスターでショートが無いか確認します。

| チェック内容 | 期待する結果 |
|---|---|
| `+5V` と `GND` の間 | **導通しない**(ショートしていない) |
| `+3V3` と `GND` の間 | **導通しない** |

**ショートしていたら通電しないでください。** はんだブリッジを探して修正します
(IC の電源ピン周辺が最も怪しいです)。

> 💡 テスターの使い方: 「導通モード(ブザー)」にして2点に当て、**ピーと鳴らなければOK**です。

---

## 5. ファームウェアを書き込む

Stamp を基板に載せ、USB-C ケーブルで PC に接続します。

```sh
cd firmware
pio run -t upload
```

**成功すると Stamp の内蔵LEDが緑色に点灯します**(= 電源OK・ファーム起動OK)。

| LED | 意味 |
|---|---|
| 🟢 緑 | 待機中(正常) |
| 🔵 青 | 吸い出し中 |
| 🔴 赤点滅 | エラー |

---

## 6. セルフテスト(カセット不要)

**https://goroman.github.io/nes-rom-reader/web/** を Chrome で開いて:

1. **接続** をクリック → ポートを選択
2. **⚙ セルフテスト(T)** をクリック

各ピンが約1秒ごとに検査され、最後に **`SELFTEST PASS`** が出れば OK です。

> ℹ️ セルフテストは ESP32 側の配線を見るもので、カセットまでの配線は検証しません。
> 最終確認は次の実カセット吸い出しで行います。

---

## 7. 実カセットで吸い出す

> ⚠️ **カセットの抜き差しは必ず電源OFF(USBを抜いた状態)で。**

![カセットを挿して吸い出し](images/assembly/07-complete.jpg)

1. カセットをスロットに**しっかり奥まで**挿す
2. USB を接続 → Web UI で **接続**
3. PRG `32` / CHR `8`(NROMの場合)を指定して **吸い出して .nes 保存**
4. 表示される **PRG-CRC / CHR-CRC** を確認

**同じカセットで2〜3回吸い出して、CRCが毎回同じなら成功です 🎉**

---

## 8. うまくいかないときは

### 🔴 PRG の CRC が毎回変わる(データがランダム)

**いちばん多い原因はカセットの接点不良です。**

1. **カセットを数回抜き差しする**(接点の酸化膜が取れます)。接点復活剤があればなお良し
2. 直らなければテスターで導通確認:
   - `/ROMSEL`: Stamp の `G41` → (U8) → スロット **44番ピン**
   - CPUデータ: スロット **36〜43番ピン** → U6
3. U6・U8(またはジャンパ)のはんだを再溶融

> 実際にこの症状が出たときの原因は、**カセットの接点とジャンパの接触不良**でした。

### 切り分けの早見表

CHR と PRG は経路が別なので、症状から原因を絞れます。

| 症状 | 疑うべき箇所 |
|---|---|
| **PRG だけ**不安定 | `/ROMSEL`(スロット44)、CPUデータ(36–43)、U6 |
| **CHR だけ**不安定 | `PPU /RD`(スロット17)、PPUデータ(26–29, 57–60)、U7 |
| **両方**ダメ | 電源、U8(またはジャンパ)、シフトレジスタ U2–U5 |

### ミラーリングが `?` になる

カセット未挿入・接触不良、または4画面ミラーリングのカセットです。
R1/R2 を付けていない場合も `?` になりますが、**吸い出し自体は可能**です。

### LED が赤点滅する

不正コマンドかセルフテスト失敗です。Web UI のログを確認してください。

### そもそも Stamp が認識されない

- USB ケーブルが**充電専用**になっていないか確認(データ対応ケーブルが必要)
- Stamp の実装(嵩上げ配線)に間違いがないか、テスターで確認

---

## 9. 完成したら

- 吸い出した `.nes` を **エミュレータ(Mesen など)で起動**して、ゲームが動けば完璧です
- CHR の中身は Web UI の **🐵 Super Monkey Viewer** で見られます(タイル表示・ドット絵編集も可能)
- 複数枚作った場合は [出荷テスト](../README.ja.md) で全数検査ができます
