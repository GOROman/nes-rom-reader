# 基板 v0.2 設計変更仕様

v0.1 は**読み出し専用**（データバッファが方向固定）のため、マッパーレジスタを書けない。
その結果、バスコンフリクトを起こさない MMC1 / MMC3 系（mapper 1, 4, 165 など）は
バンク選択ができず、ROM全体を吸い出せない。v0.2 はここを解消する。

あわせて v0.1 で判明した実装上の不具合（M5Stamp S3 が挿さらない）も修正する。

## 変更点サマリ

| # | 項目 | v0.1 | v0.2 | 理由 |
|---|---|---|---|---|
| 1 | データバッファ U6/U7 | 74LVC245（DIR固定 B→A） | **SN74LVC8T245PWR**（TSSOP-24、DIR制御） | カートリッジへ書けるようにする |
| 2 | バス方向制御 | なし | **BUS_DIR = G40**（U6/U7 共通1本） | マッパーレジスタ書き込み |
| 3 | /OE プルアップ | なし | OE_PRG_N / OE_CHR_N に 10k → +3V3 | MCU起動中にバスを駆動しないため（必須） |
| 4 | U1 フットプリント | 寸法誤りで挿さらない | 実測に合わせて修正 | v0.1 既知不具合 |
| 5 | U8 (74HCT541) | ライブラリのパッド欠落品を手修正 | 検証済みSOIC-20部品に差し替え | 再発防止 |

## 1. データバッファの置き換え

**SN74LVC8T245PWR** は VCCA=3.3V / VCCB=5V のデュアル電源レベル変換トランシーバ。
制御ピン（DIR, /OE）は **VCCA(3.3V) 基準**なので、ESP32-S3 から直接駆動できる。

| ピン | 接続 |
|---|---|
| VCCA | +3V3 |
| VCCB | +5V |
| A1–A8 | MCU 側データバス MD0–MD7 (G4–G11) |
| B1–B8 | カートリッジ側データバス |
| DIR | BUS_DIR (G40)。**H = A→B（MCUがカートを駆動 = 書き込み）**、L = B→A（読み出し） |
| /OE | U6=OE_PRG_N (G15) / U7=OE_CHR_N (G39) |

74LVC245（VCC単一3.3V・5Vトレラント）では A→B 側の出力が 3.3V にしかならず、
5V CMOS のマッパーIC には VIH が足りない。デュアル電源品にするのはこのため。

## 2. BUS_DIR は 1 本で足りる

U6（CPUデータ）と U7（PPUデータ）で DIR を共有してよい。
どちらのバッファが実際にバスへ出るかは **/OE が独立に決めている**ため、
書き込み中に使わない側は /OE=H で Hi-Z のままとなり干渉しない。

→ 追加で必要な GPIO は **G40 の 1 本だけ**。他のピンアサインは v0.1 から変更なし。

## 3. /OE プルアップ（安全上の必須項目）

ESP32-S3 のリセット〜ファーム起動までの間、GPIO は入力（Hi-Z）になる。
このとき /OE がフロートすると、DIR の状態次第でバッファがカートリッジのバスを
駆動してしまい、カートリッジ側 ROM と衝突する可能性がある。
OE_PRG_N / OE_CHR_N に **10kΩ プルアップ（→ +3V3）** を入れ、
起動中は必ず Hi-Z になるようにする。

BUS_DIR 側は 10kΩ プルダウン（→ GND、読み出し方向）にしておくと更に安全。

## 4. 追加BOM（v0.1 からの差分）

| 部品 | リファレンス | 数 | 備考 |
|---|---|---|---|
| SN74LVC8T245PWR (TSSOP-24) | U6, U7 | 2 | 74LVC245 と置き換え。ピン数・フットプリントが変わる |
| 抵抗 10kΩ | R3, R4 | 2 | OE_PRG_N / OE_CHR_N プルアップ |
| 抵抗 10kΩ | R5 | 1 | BUS_DIR プルダウン |

## 5. これで対応できるようになるもの

- **MMC1 (mapper 1)** — シリアル書き込み（5回に分けて$8000-$FFFFへ書く）でPRG/CHRバンク選択
- **MMC3 (mapper 4)** — $8000/$8001 のレジスタ書き込みでバンク選択、$A001 で WRAM 有効化
- **mapper 165 等のMMC3派生** — 同上（*Fire Emblem (Unl)* など）
- **UNROM (mapper 2)** — バスコンフリクトを気にせず確実にバンク選択
- **コピープロテクトダイオード付きCNROM** — バスを実際に駆動して勝てる
- **CHR-RAM への書き込み** — Super Monkey Viewer で編集した内容をカセットへ戻す（PPU /WR は配線済み）

## 6. ファームウェア側

`BOARD_REV` で切り替える。**v0.1 基板で書き込み機能を有効にしてはいけない**
（バッファが B→A 固定のため、MCU の出力と 245 の出力が衝突してショートする）。

```ini
; platformio.ini
build_flags = -D BOARD_REV=2   ; v0.1 基板では 1 のまま
```

追加コマンド（BOARD_REV=2 のときのみ有効、v0.1 では `ERR` を返す）:

| コマンド | 意味 |
|---|---|
| `P <addr_hex> <data_hex>` | CPU空間へ1バイト書き込み（マッパーレジスタ／WRAM） |
